# Configuration

SPECTRA's configuration, environment variables, and project directory structure.

## config.yaml

Located at `.spectra/config.yaml`. Created by `spectra init`.

```yaml
spectra:
  version: "1.0"
  project_id: "my-project"

ai:
  adapter: "none"
  primary_agent: "none"

spec:
  id_prefix: "feat"
  default_status: "draft"

git:
  gate_branch_protection: false
  trace_commit_hook: false

hooks: {}
```

### Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `spectra.version` | string | `"1.0"` | Config schema version |
| `spectra.project_id` | string | `"my-project"` | Project identifier |
| `ai.adapter` | string | `"none"` | AI adapter (future: `openai`, `anthropic`, etc.) |
| `ai.primary_agent` | string | `"none"` | Primary AI agent for generation |
| `spec.id_prefix` | string | `"feat"` | Default prefix for new spec IDs |
| `spec.default_status` | string | `"draft"` | Default status for new specs |
| `git.gate_branch_protection` | boolean | `false` | Enable branch protection based on gates (future) |
| `git.trace_commit_hook` | boolean | `false` | Enable trace comment validation on commit (future) |
| `hooks` | object | `{}` | Lifecycle hooks (future) |

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `SPECTRA_INSTALL_DIR` | `~/.spectra-sdd` | Installation directory (used by `install.sh`) |
| `SPECTRA_BIN_DIR` | auto-detected | Binary symlink directory (used by `install.sh`) |
| `NO_COLOR` | -- | Disable colored output in install/uninstall scripts |

## .spectra/ Directory Layout

```
.spectra/
  config.yaml                  Project configuration
  constitution.yaml            Constitutional constraints (Tier 0)
  constitution.changelog       Immutable amendment audit log (append-only)
  trace.json                   Traceability matrix
  generate.lock                Generation lock file (input/output hash cache)
  .gitignore                   Ignores drift.json

  features/
    _index.yaml                Progressive disclosure index (auto-rebuilt)
    *.spec.yaml                Feature specs (Tier 1)

  impl/
    <feature-name>/
      *.impl.yaml              Implementation specs (Tier 2), one per concern

  tests/
    *.test.yaml                Test specs (Tier 3)

  migrations/
    *.migration.yaml           Migration specs (Tier 4)

  gates/
    *.gate.yaml                Gate files, one per spec+phase
                               Naming: <safeId>@<semver>--<phase>.gate.yaml

  templates/
    *.tmpl                     Project-local Handlebars template overrides

  adapters/
    (empty)                    AI adapter configurations (future)
```

### File Details

**config.yaml** -- Validated against `ConfigSchema`. Falls back to defaults on parse error.

**constitution.yaml** -- Validated against `ConstitutionSchema`. Defines vocabulary and MUST/SHOULD/MAY constraints.

**constitution.changelog** -- Plain text, append-only. Each entry records: timestamp, action, author, description, previous hash, new hash, approvers.

**trace.json** -- Validated against `TraceMatrixSchema`. Maps spec IDs to authorized artifacts, AC coverage, and gate status.

**generate.lock** -- JSON object keyed by `<specId>@<specVersion>--<target>`. Each entry stores template hash, input hash, output hash, generation ID, and timestamp.

**_index.yaml** -- Validated against `SpecIndexSchema`. Lightweight cache of all feature spec metadata. Auto-rebuilt on `spec new`, `spec rehash`, and `trace update`.

**drift.json** -- Generated by `spectra diff --save`. Gitignored by default (volatile computed artifact).

## Brownfield vs Greenfield

| | Greenfield (`spectra init`) | Brownfield (`spectra init --brownfield`) |
|---|---|---|
| Constitution version | `1.0.0` | `0.1.0` |
| Constitution status | `active` | Provisional |
| Changelog note | "Initial constitution" | "Provisional — constraints being discovered" |
| Use case | New project | Existing codebase adopting SDD |
